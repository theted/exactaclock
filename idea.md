This is **Exactaclock** - it displays the "corrected" GMT time for given GPS coordinates - in other words, if a coordinate is exactly between two timezones, the difference will be 30 min (in normal cases), if closer to the timezone border, the difference will be less, etc. Calculations should take both horizontal and vertical distance into account, and display the corrected time.

- The application should be a served as a simple single-page HTML application, served on the client side. vanilla ES6 javascript can be used, no need for any frameworks. Prefer arrow functions. Should be kept as simple as possible. May include a build step to keep development code as clean as possible.
- An getExacTime() function should take a GPS coordinate, and determine the corrected UTC time, depending on how close the coordinates are to it's closest timezones. internally, this function might call getTimezone(coordinates) and getClosestTimezones(coordinates) or similarly named internal helper functions, to keep the code logic cleaner. These will need to be implemented. A list of timezones will also need to be included.
- The web application will be a simple but clean UI: it should ask the user for permission to get the device GPS location, then display 'Current exact GMT time' with the corrected GMT time in big bold letters. This should work as actual clock, e.g. update the time every second. The user's location, timezone and corrected local time should also be displayed.
